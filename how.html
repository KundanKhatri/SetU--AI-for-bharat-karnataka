How Voca works

Most voice AI: hear → respond.
Voca: hear → restate → verify → only then act.

The verification turn is not a usability feature. It is the structural rule of the system. No interpretation, however confident, is acted upon without explicit citizen confirmation. There is one exception, and it is the most important rule of the product.

Voca moves through these four states for every utterance. Only the Verified state ever triggers a real-world action. The other states are auditable, reversible, and discardable.
Each state, in words
What happens, what doesn't.
State 01

Heard

The citizen has spoken. Their audio has been captured by the SIP gateway, passed through streaming STT, and converted into text with an n-best list and per-token confidence.

What this state cannot do: nothing actionable. Heard is just a buffer. We do not call any service, route any request, or commit any state change off Heard alone.

State 02

Interpreted

The interpretation LLM has produced a structured grievance record from the transcript: {intent, entities, risk_signal, sentiment, confidence, language, paraphrase, agent_suggestion}.

What this state cannot do: still nothing actionable. A 0.99-confidence interpretation has the same actionability as a 0.30 one — zero. The point of Interpreted is to prepare a verification turn, not to commit to a decision.

State 03

Verified

Voca has spoken the paraphrase back to the citizen, in their language. The citizen has explicitly confirmed: "yes, that's right." The classifier has parsed their response as correct (vs. partial or incorrect).

What this state can do: everything. Verified is the gate. Routing, dispatching, ticketing, escalation, audit-log commits — all of it begins here.

State 04

Acted

The action has been taken. The grievance has been routed to the right department. The ambulance has been dispatched. The agent has been briefed. An immutable audit log entry now exists, retrievable per call ID in ≤ 5 seconds.

What this state means for review: every action in the system can be traced back through Verified → Interpreted → Heard with full timestamps, model versions, and payload hashes. An IAS officer auditing the call sees exactly what happened, in order.

The exception
When seconds matter more than verification.

There is one situation where the verify-before-act rule is consciously suspended. It is the most important rule of the product.

If distress and risk-signal both exceed threshold —
Voca bypasses verification.

Specifically: when the sentiment engine reports distress greater than 0.65 and the interpretation LLM marks the risk_signal as HIGH, the FSM commits to one rule:

Skip verification. Speak rapid-acknowledgement in the citizen's dialect — "I hear you. Help is coming." — and route to a human agent in parallel.

The agent picks up at t = 2.5 s. The citizen never holds an unanswered line. The audit log captures the bypass with the threshold values and the trigger phrase. The system's own behavior, recorded.

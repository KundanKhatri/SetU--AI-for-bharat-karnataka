Setu — the AI verify-before-act bridge for the 1092 helpline

A voice-to-voice assistive layer for Karnataka's citizen helpline. Hears the caller in their dialect, restates back what it understood, only then acts. Hands off gracefully — with full context — when confidence drops or distress spikes.

Ingress (audio in) Brain (interpret + feel) Verification FSM Egress (voice out) Agent console Storage + flywheel

02 The four-state lifecycle (the load-bearing rule)

State 1 Heard
Audio captured, STT produced text + n-best alternatives + per-token confidence. No interpretation yet.
State 2 Interpreted
LLM produced an {intent, entities, sentiment, confidence} tuple. Still not actionable.
State 3 Verified
Citizen has explicitly confirmed the interpretation in their own language. Only this state is actionable.
State 4 Acted
Routed to agent / KB / structured action. Audit event written.
Verification patterns by confidence
ConfidencePattern
≥ 0.85Single-shot paraphrase: "Just to confirm — you're saying X. Is that right?"
0.6 – 0.85Open-ended: "You're calling because Y. Could you say that's correct, or tell me what I missed?"
< 0.6No paraphrase. One targeted clarifying question. After 2 misses → human takeover.
Bypass rule: distress > 0.75 AND risk_signal == HIGH → suppress verification, do rapid acknowledgement, parallel handover.

03 Three-layer dialect handling

Layer 1 · Acoustic
AI4Bharat IndicConformer-multilingual STT. 22 Indic languages, dialect-tagged training data for Kannada (Mandya, Belagavi, Mangaluru), Hindi, Telugu. n-best hypotheses preserved — disambiguation is deferred to the LLM with surrounding context.
Layer 2 · Lexical
Sarvam-1 (or Llama-3.3 8B Indic-tuned) fine-tuned on a curated grievance corpus. Karnataka place names, department names, dialect-paired sentence pairs. Indic-native tokenizer (3–4× more efficient than GPT tokenizer on Devanagari/Kannada script).
Layer 3 · Cultural
Honorific normalizer (neevu / neenu / taavu), idiom dictionary (curated, not generated), district-entity retrieval (when caller says "galli", retrieve their stated location). Deterministic — belongs in code + KB, not in a stochastic model.

04 Sentiment as a routing signal

SignalDetectionSystem behavior
Distress + urgency highProsody (pitch jitter, energy spike) + lexical risk-wordsBypass full verification · rapid-ack · parallel handover
Sustained confusionContinuous low-confidence STT + slow turn-takingSwitch to simpler register · slower TTS pace · shorter sentences
Anger at the systemLexical anger toward "you/AI/robot" + escalating volumeImmediate handover · no placation loop · full context to agent
Fear escalatingProsodic fear signature growing across turnsSoft handover trigger · agent prepared · empathetic acknowledgement
Neutral / calmDefault stateStandard verification flow
